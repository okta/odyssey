# Generating Odyssey Icons

## tl;dr

1. Designer exports all icons as SVGs.
2. Put icons in `packages/core/odyssey-icons/src/figma.icon.generated` and logos in `packages/core/odyssey-icons/src/figma.logo.generated`.
3. Run `yarn generate:icons` from the project root.

## Icon types

Odyssey icons are split into two categories:

- **icons** — general-purpose UI icons (arrows, actions, status indicators, etc.)
- **logos** — authenticator and service brand logos (Duo, Google Authenticator, Okta Verify, Symantec VIP, YubiKey)

## Verbose

When we have new icons in Figma, a designer will add those to the `packages/core/odyssey-icons` package in the appropriate `src/figma.*.generated/` subfolder.
**Note:** SVGs must be exported at the correct size: **16×16 for icons** and **32×32 for logos**.

After that's complete, run this command to generate our icons:

```sh
yarn generate:icons
```

This runs the following steps automatically:

1. **Build** `odyssey-icons` to produce optimized SVGs in `dist/icons` (system) and `dist/logos` (logos).
2. **Generate system components** — SVGR converts `dist/icons/*.svg` into React components under `odyssey-react-mui/src/icons.generated/`.
3. **Generate logo components** — SVGR converts `dist/logos/*.svg` into React components under `odyssey-react-mui/src/logos.generated/`.
4. **Generate the index** — `scripts/generateIconsIndex.ts` scans both output directories and writes `odyssey-react-mui/src/icons.generated/index.ts` with re-exports for every generated component. No manual edits to the index file are needed.

After icons are generated, add stories for any new icons in:

- `packages/apps/odyssey-storybook/src/Odyssey Core/Icon/Icon.stories.tsx`
