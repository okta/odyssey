# Globalization within Odyssey

Odyssey has its own globalization system which other devs can hook into.

## OdysseyTranslationProvider

Odyssey comes out of the box with support for the following 30 languages:

```ts

  "cs", // Czech
  "da", // Danish
  "de", // German
  "el", // Greek
  "en", // English
  "es", // Spanish
  "fi", // Finnish
  "fr", // French
  "ht", // Haitian Creole
  "hu", // Hungarian
  "id", // Indonesian
  "it", // Italian
  "ja", // Japanese
  "ko", // Korean
  "ms", // Malaysian
  "nb", // Norwegian
  "nl_NL", // Dutch
  "ok_PL", // (Test Language) Pseudo-loc
  "ok_SK", // (Test Language) Show Keys
  "pl", // Polish
  "pt_BR", // Portuguese (Brazil)

  "ru", // Russian
  "sv", // Swedish

  "tr", // Turkish
  "uk", // Ukrainian
  "vi", // Vietnamese
  "zh_CN", // Chinese (PRC)
  "zh_TW", // Chinese
] as const;
```

### Provider customization

By default, it will use the `window.navigator.language` (browser setting) or `en` as the selected language code. It is possible to provide `languageCode` to the provider via props:

```jsx
import { OdysseyTranslationProvider } from "@okta/odyssey-react-mui";

const languageCode = "cs";

const YourAppRoot = ({ children }) => (
  <OdysseyTranslationProvider languageCode={languageCode}>
    {children}
  </OdysseyTranslationProvider>
);
```

If you want to override translation bundles, you can do so by providing the `translationOverrides` prop as such:

```jsx
import { OdysseyTranslationProvider } from "@okta/odyssey-react-mui";

const translationOverrides = {
  en: {
    "fieldlabel.optional.text": "Not Required",
  },
  fr: {
    "fieldlabel.optional.text": "Non Requis",
  },
};

const YourAppRoot = ({ children }) => (
  <OdysseyTranslationProvider languageCode={languageCode}>
    {children}
  </OdysseyTranslationProvider>
);
```

## Passing the language from the Monolith

```
import { getOktaLocale } from '@okta/ui-libraries-monolith';

<OdysseyProvider languageCode={getOktaLocale()}>
  <CssBaseline>

    // Add code here

  </CssBaseline>
</OdysseyProvider>

```
